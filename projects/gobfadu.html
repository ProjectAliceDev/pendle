The SIP of the Ren'Py Operating System

The GOBFADU Policy (Gobrieno Federal de Aduana Mexico Policy) is a SIP (System Integrity Protection) used in AliceOS. It's job is to enforce a Security Policy built to prevent unauthorized modification and tampering to both the AliceOS Native OS and the AliceOS Kernel.

Built with Applet Security in Mind

Applets make up the app ecosystem for AliceOS. Applets are how third-party developers add additional functionality to AliceOS and interact with users and players. For Applet Security, the GOBFADU Policy checks for any modification that is unauthorized and instantly blocks the Applet from running on the OS, prompting you to reinstall the Applet entirely.

Notification Control

AliceOS comes with its own notification framework that allows developers and users to stay informed. Applets and system services alike use notifications to express ideas, display information, or even make a request. Notifications are also user-controlled, giving the user ultimate control over which notifications can be seen.

Privacy Control

Privacy and security matter to everyone. That's why AliceOS comes with its own security policy, provided under the GOBFADU policy. Files have integrity protection and are not allowed to be tampered with. Furthermore, we enforce a policy to ensure that only you give the consent for an applet's control to things like notifications, file system access, and system setting changes.

Let's get geeky

AliceOS provides an experience similar to Linux, complete with a Terminal and a UNIX-like shell. From basic filesystem commands to package installation, AliceOS provides a unique experience to games. Applets can harness the power of the Terminal and offer additional commands and tools through its command-line interface, perfect for any character deletion (we're looking at you, Monika).
